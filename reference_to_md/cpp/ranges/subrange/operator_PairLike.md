### [[edit](https://en.cppreference.com/mwiki/index.php?title=cpp/ranges/subrange/operator_PairLike&action=edit&section=2 "Edit section: Notes")] Notes

Following types in the standard library are pair-like: 

  * [std::pair](../../utility/pair.html)<T, U>
  * [std::tuple](../../utility/tuple.html)<T, U>
  * [std::array](../../container/array.html)<T, 2>
  * std::[ranges::subrange](../subrange.html)<I, S, K>



  * [std::complex](../../numeric/complex.html)<T>

| (since C++26)  
---|---  
  
  


A program-defined type derived from one of these types can be a pair-like type, if 

  * [`std::tuple_size`](../../utility/tuple_size.html "cpp/utility/tuple size") and [`std::tuple_element`](../../utility/tuple_element.html "cpp/utility/tuple element") are correctly specialized for it, and 
  * calls to std::get<0> and std::get<1> for its value are well-formed. 

| (until C++23)  
---|---  
  
Since `subrange` specializations are [`range`](../range.html "cpp/ranges/range") types, conversion to them are not performed via this conversion function. 

[std::array](../../container/array.html "cpp/container/array") specializations cannot be converted from `subrange`, since they are [`range`](../range.html "cpp/ranges/range") types. 

### [[edit](https://en.cppreference.com/mwiki/index.php?title=cpp/ranges/subrange/operator_PairLike&action=edit&section=3 "Edit section: Example")] Example

Run this code
    
    
    #include <iostream>
    #include <ranges>
    #include <string>
    #include <utility>
     
    using striter = std::string::const_iterator;
     
    using legacy_strview = [std::pair](../../utility/pair.html)<striter, striter>;
     
    void legacy_print(legacy_strview p)
    {
        for (; p.first != p.second; ++p.first)
            [std::cout](../../io/cout.html) << *p.first << ' ';
        [std::cout](../../io/cout.html) << '\n';
    }
     
    int main()
    {
        [std::string](../../string/basic_string.html) dat{"ABCDE"};
        for (auto v{std::[ranges::subrange](../subrange.html){dat}}; v; v = {v.begin(), v.end() - 1})
        {
            /*...*/
            legacy_print(legacy_strview{v});
        }
    }

Output: 
    
    
    A B C D E 
    A B C D 
    A B C 
    A B 
    A
