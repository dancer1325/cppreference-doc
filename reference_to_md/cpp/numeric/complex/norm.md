### Parameters

z  |  \-  |  complex value   
---|---|---  
f  |  \-  |  floating-point value   
i  |  \-  |  integer value   
  
### Return value

1) The squared magnitude of z.

A) The square of f.

B) The square of i.

### Notes

The norm calculated by this function is also known as [field norm](https://en.wikipedia.org/wiki/Field_norm "enwiki:Field norm") or [absolute square](https://mathworld.wolfram.com/AbsoluteSquare.html). 

The [Euclidean norm](https://en.wikipedia.org/wiki/Euclidean_space#Euclidean_norm "enwiki:Euclidean space") of a complex number is provided by [`std::abs`](abs.html "cpp/numeric/complex/abs"), which is more costly to compute. In some situations, it may be replaced by `std::norm`, for example, if abs(z1) > abs(z2) then norm(z1) > norm(z2). 

The additional overloads are not required to be provided exactly as (A,B). They only need to be sufficient to ensure that for their argument num: 

  * If num has a standard(until C++23) floating-point type `T`, then std::norm(num) has the same effect as std::norm([std::complex](../complex.html)<T>(num)). 
  * Otherwise, if num has an integer type, then std::norm(num) has the same effect as std::norm([std::complex](../complex.html)<double>(num)). 



### Example

Run this code
    
    
    #include <cassert>
    #include <complex>
    #include <iostream>
    Â 
    int main()
    {
        constexpr [std::complex](../complex.html)<double> z {3.0, 4.0};
        static_assert(std::norm(z) == (z.real() * z.real() + z.imag() * z.imag()));
        static_assert(std::norm(z) == (z * [std::conj](conj.html)(z)));
               [assert](../../error/assert.html)(std::norm(z) == (std::abs(z) * std::abs(z)));
        [std::cout](../../io/cout.html) << "std::norm(" << z << ") = " << std::norm(z) << '\n';
    }

Output: 
    
    
    std::norm((3,4)) = 25
