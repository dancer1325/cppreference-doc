### [[edit](https://en.cppreference.com/mwiki/index.php?title=cpp/numeric/complex/tanh&action=edit&section=2 "Edit section: Return value")] Return value

If no errors occur, complex hyperbolic tangent of z is returned. 

### [[edit](https://en.cppreference.com/mwiki/index.php?title=cpp/numeric/complex/tanh&action=edit&section=3 "Edit section: Error handling and special values")] Error handling and special values

Errors are reported consistent with [math_errhandling](../math/math_errhandling.html "cpp/numeric/math/math errhandling"). 

If the implementation supports IEEE floating-point arithmetic, 

  * [std::tanh](../math/tanh.html)([std::conj](conj.html)(z)) == [std::conj](conj.html)([std::tanh](../math/tanh.html)(z)). 
  * [std::tanh](../math/tanh.html)(-z) == -[std::tanh](../math/tanh.html)(z). 
  * If z is `(+0,+0)`, the result is `(+0,+0)`. 
  * If z is `(x,+∞)` (for any[[1]](tanh.html#cite_note-1) finite x), the result is `(NaN,NaN)` and [FE_INVALID](../fenv/FE_exceptions.html "cpp/numeric/fenv/FE exceptions") is raised. 
  * If z is `(x,NaN)` (for any[[2]](tanh.html#cite_note-2) finite x), the result is `(NaN,NaN)` and [FE_INVALID](../fenv/FE_exceptions.html "cpp/numeric/fenv/FE exceptions") may be raised. 
  * If z is `(+∞,y)` (for any finite positive y), the result is `(1,+0)`. 
  * If z is `(+∞,+∞)`, the result is `(1,±0)` (the sign of the imaginary part is unspecified). 
  * If z is `(+∞,NaN)`, the result is `(1,±0)` (the sign of the imaginary part is unspecified). 
  * If z is `(NaN,+0)`, the result is `(NaN,+0)`. 
  * If z is `(NaN,y)` (for any non-zero y), the result is `(NaN,NaN)` and [FE_INVALID](../fenv/FE_exceptions.html "cpp/numeric/fenv/FE exceptions") may be raised. 
  * If z is `(NaN,NaN)`, the result is `(NaN,NaN)`. 


  1. [↑](tanh.html#cite_ref-1) per [C11 DR471](https://open-std.org/JTC1/SC22/WG14/www/docs/n1892.htm#dr_471), this only holds for non-zero x. If `z` is `(0,∞)`, the result should be `(0,NaN)`.
  2. [↑](tanh.html#cite_ref-2) per [C11 DR471](https://open-std.org/JTC1/SC22/WG14/www/docs/n1892.htm#dr_471), this only holds for non-zero x. If `z` is `(0,NaN)`, the result should be `(0,NaN)`.



### [[edit](https://en.cppreference.com/mwiki/index.php?title=cpp/numeric/complex/tanh&action=edit&section=4 "Edit section: Notes")] Notes

Mathematical definition of hyperbolic tangent is tanh z = ez  
-e-z  
  
---  
ez  
+e-z  
  
. 

Hyperbolic tangent is an analytical function on the complex plane and has no branch cuts. It is periodic with respect to the imaginary component, with period πi, and has poles of the first order along the imaginary line, at coordinates (0, π(1/2 + n)). However no common floating-point representation is able to represent π/2 exactly, thus there is no value of the argument for which a pole error occurs. 

### [[edit](https://en.cppreference.com/mwiki/index.php?title=cpp/numeric/complex/tanh&action=edit&section=5 "Edit section: Example")] Example

Run this code
    
    
    #include <cmath>
    #include <complex>
    #include <iostream>
     
    int main()
    {
        [std::cout](../../io/cout.html) << [std::fixed](../../io/manip/fixed.html);
        [std::complex](../complex.html)<double> z(1.0, 0.0); // behaves like real tanh along the real line
        [std::cout](../../io/cout.html) << "tanh" << z << " = " << [std::tanh](../math/tanh.html)(z)
                  << " (tanh(1) = " << [std::tanh](../math/tanh.html)(1) << ")\n";
     
        [std::complex](../complex.html)<double> z2(0.0, 1.0); // behaves like tangent along the imaginary line
        [std::cout](../../io/cout.html) << "tanh" << z2 << " = " << [std::tanh](../math/tanh.html)(z2)
                  << " ( tan(1) = " << [std::tan](../math/tan.html)(1) << ")\n";
    }

Output: 
    
    
    tanh(1.000000,0.000000) = (0.761594,0.000000) (tanh(1) = 0.761594)
    tanh(0.000000,1.000000) = (0.000000,1.557408) ( tan(1) = 1.557408)
