**tuple_element <std::complex>**(C++26)  
  


Defined in header `[<complex>](../../header/complex.html "cpp/header/complex")` |  |   
---|---|---  
template< [std::size_t](../../types/size_t.html) I, class T >  
struct tuple_element<I, [std::complex](../complex.html)<T>>; |  |  (since C++26)  
| |   
  
The partial specializations of [`std::tuple_element`](../../utility/tuple_element.html "cpp/utility/tuple element") for [std::complex](../complex.html "cpp/numeric/complex") provide compile-time access to the underlying real and imaginary number type of a `complex`, using tuple-like syntax. They are provided for structured binding support. The program is ill-formed if I >= 2.

### Example

Run this code
    
    
    #include <complex>
    #include <type_traits>
     
    static_assert([z = [std::complex](../complex.html)<float>()]
    {
        using T = decltype(z);
        return
    #if __cpp_lib_tuple_like >= 202311L
            [std::is_same_v](../../types/is_same.html)<[std::tuple_element_t](../../utility/tuple_element.html)<0, T>, float> &&
            [std::is_same_v](../../types/is_same.html)<[std::tuple_element_t](../../utility/tuple_element.html)<1, T>, float> &&
    #endif
            [std::is_same_v](../../types/is_same.html)<T::value_type, float>;
    }());
     
    int main() {}
