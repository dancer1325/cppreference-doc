1-4) Computes complex x raised to a complex power y with a branch cut along the negative real axis for the first argument. Non-complex arguments are treated as complex numbers with positive zero imaginary component.

A-C) Additional overloads are provided. `NonComplex` is not a specialization of [std::complex](../complex.html "cpp/numeric/complex"). | (since C++11)

### Parameters

x  |  \-  |  base   
---|---|---  
y  |  \-  |  exponent   
  
### Return value

1-4) If no errors occur, the complex power xy  
, is returned.

Errors and special cases are handled as if the operation is implemented by [std::exp](../math/exp.html)(y * [std::log](../math/log.html)(x)).

The result of [std::pow](../math/pow.html)(0, 0) is implementation-defined.

A-C) Same as (2-4).

### Notes

Overload (1) was provided in C++98 to match the extra overloads (2) of [std::pow](../math/pow.html "cpp/numeric/math/pow"). Those overloads were removed by the resolution of [LWG issue 550](https://cplusplus.github.io/LWG/issue550), and overload (1) was removed by the resolution of [LWG issue 844](https://cplusplus.github.io/LWG/issue844). 

The additional overloads are not required to be provided exactly as (A-C). They only need to be sufficient to ensure that for their first argument base and second argument exponent: 

If base and/or exponent has type [std::complex](../complex.html)<T>: 

  * If base and/or exponent has type [std::complex](../complex.html)<long double> or long double, then `std::pow(base, exponent)` has the same effect as `std::pow`([std::complex](../complex.html)<long double>(base),  
[std::complex](../complex.html)<long double>(exponent)). 
  * Otherwise, if base and/or exponent has type [std::complex](../complex.html)<double>, double, or an integer type, then `std::pow(base, exponent)` has the same effect as `std::pow`([std::complex](../complex.html)<double>(base),  
[std::complex](../complex.html)<double>(exponent)). 
  * Otherwise, if base and/or exponent has type [std::complex](../complex.html)<float> or float, then `std::pow(base, exponent)` has the same effect as `std::pow`([std::complex](../complex.html)<float>(base),  
[std::complex](../complex.html)<float>(exponent)). 

| (until C++23)  
---|---  
If one argument has type [std::complex](../complex.html)<T1> and the other argument has type `T2` or [std::complex](../complex.html)<T2>, then `std::pow(base, exponent)` has the same effect as `std::pow`([std::complex](../complex.html)<[std::common_type_t](../../types/common_type.html)<T1, T2>>(base),  
[std::complex](../complex.html)<[std::common_type_t](../../types/common_type.html)<T1, T2>>(exponent)). If [std::common_type_t](../../types/common_type.html)<T1, T2> is not well-formed, then the program is ill-formed.  | (since C++23)  
  
### Example

Run this code
    
    
    #include <complex>
    #include <iostream>
     
    int main()
    {
        [std::cout](../../io/cout.html) << [std::fixed](../../io/manip/fixed.html);
     
        [std::complex](../complex.html)<double> z(1.0, 2.0);
        [std::cout](../../io/cout.html) << "(1,2)^2 = " << [std::pow](../math/pow.html)(z, 2) << '\n';
     
        [std::complex](../complex.html)<double> z2(-1.0, 0.0); // square root of -1
        [std::cout](../../io/cout.html) << "-1^0.5 = " << [std::pow](../math/pow.html)(z2, 0.5) << '\n';
     
        [std::complex](../complex.html)<double> z3(-1.0, -0.0); // other side of the cut
        [std::cout](../../io/cout.html) << "(-1,-0)^0.5 = " << [std::pow](../math/pow.html)(z3, 0.5) << '\n';
     
        [std::complex](../complex.html)<double> i(0.0, 1.0); // i^i = exp(-pi / 2)
        [std::cout](../../io/cout.html) << "i^i = " << [std::pow](../math/pow.html)(i, i) << '\n';
    }

Output: 
    
    
    (1,2)^2 = (-3.000000,4.000000)
    -1^0.5 = (0.000000,1.000000)
    (-1,-0)^0.5 = (0.000000,-1.000000)
    i^i = (0.207880,0.000000)
