int fseek( [std::FILE](FILE.html)* stream, long offset, int origin ); |  |   
| |   
  
Sets the file position indicator for the file stream stream. 

If the stream is open in binary mode, the new position is exactly offset bytes measured from the beginning of the file if origin is [SEEK_SET](../c.html "cpp/io/c"), from the current file position if origin is [SEEK_CUR](../c.html "cpp/io/c"), and from the end of the file if origin is [SEEK_END](../c.html "cpp/io/c"). Binary streams are not required to support [SEEK_END](../c.html "cpp/io/c"), in particular if additional null bytes are output. 

If the stream is open in text mode, the only supported values for offset are zero (which works with any origin) and a value returned by an earlier call to [std::ftell](ftell.html "cpp/io/c/ftell") on a stream associated with the same file (which only works with origin of [SEEK_SET](../c.html "cpp/io/c")). 

If the stream is wide-oriented, the restrictions of both text and binary streams apply (result of [std::ftell](ftell.html "cpp/io/c/ftell") is allowed with [SEEK_SET](../c.html "cpp/io/c") and zero offset is allowed from [SEEK_SET](../c.html "cpp/io/c") and [SEEK_CUR](../c.html "cpp/io/c"), but not [SEEK_END](../c.html "cpp/io/c")). 

In addition to changing the file position indicator, `fseek` undoes the effects of [std::ungetc](ungetc.html "cpp/io/c/ungetc") and clears the end-of-file status, if applicable. 

If a read or write error occurs, the error indicator for the stream ([std::ferror](ferror.html "cpp/io/c/ferror")) is set and the file position is unaffected.

stream  |  \-  |  file stream to modify   
---|---|---  
offset  |  \-  |  number of characters to shift the position relative to origin   
origin  |  \-  |  position to which offset is added. It can have one of the following values: [SEEK_SET](../c.html "cpp/io/c"), [SEEK_CUR](../c.html "cpp/io/c"), [SEEK_END](../c.html "cpp/io/c")  
  
### [[edit](https://en.cppreference.com/mwiki/index.php?title=cpp/io/c/fseek&action=edit&section=2 "Edit section: Return value")] Return value

​0​ upon success, nonzero value otherwise. 

### [[edit](https://en.cppreference.com/mwiki/index.php?title=cpp/io/c/fseek&action=edit&section=3 "Edit section: Notes")] Notes

After seeking to a non-end position in a wide stream, the next call to any output function may render the remainder of the file undefined, e.g. by outputting a multibyte sequence of a different length. 

POSIX allows seeking beyond the existing end of file. If an output is performed after this seek, any read from the gap will return zero bytes. Where supported by the filesystem, this creates a _sparse file_. 

POSIX also requires that `fseek` first performs [fflush](fflush.html "cpp/io/c/fflush") if there are any unwritten data (but whether the shift state is restored is implementation-defined). The standard C++ file streams guarantee both flushing and unshifting: [std::basic_filebuf::seekoff](../basic_streambuf/pubseekoff.html "cpp/io/basic streambuf/pubseekoff"). 

POSIX specifies, that [`fseek`](https://pubs.opengroup.org/onlinepubs/9699919799/functions/fseek.html) should return -1 on error, and set [errno](../../error/errno.html "cpp/error/errno") to indicate the error. 

On Windows, [`_fseeki64`](https://learn.microsoft.com/en-us/cpp/c-runtime-library/reference/fseek-fseeki64) can be used to work with files larger than 2 GiB. 

### [[edit](https://en.cppreference.com/mwiki/index.php?title=cpp/io/c/fseek&action=edit&section=4 "Edit section: Example")] Example

Run this code
    
    
    #include <cassert>
    #include <cstdio>
    #include <cstdint>
    #include <fstream>
    #include <vector>
     
    int main()
    {
        [std::ofstream](../basic_ofstream.html)("dummy.nfo") << "8 bytes\n"; // create the file
     
        [std::FILE](FILE.html)* fp = [std::fopen](fopen.html)("dummy.nfo", "rb");
        [assert](../../error/assert.html)(fp);
     
        std::fseek(fp, 0, [SEEK_END](../c.html)); // seek to end
        const [std::size_t](../../types/size_t.html) filesize = [std::ftell](ftell.html)(fp);
        [std::vector](../../container/vector.html)<[std::uint8_t](../../types/integer.html)> buffer(filesize);
     
        std::fseek(fp, 0, [SEEK_SET](../c.html)); // seek to start
        [std::fread](fread.html)(buffer.data(), sizeof([std::uint8_t](../../types/integer.html)), buffer.size(), fp);
     
        [std::fclose](fclose.html)(fp);
        [std::printf](fprintf.html)("I've read %zi bytes\n", filesize);
    }

Possible output: 
    
    
    I've read 8 bytes
