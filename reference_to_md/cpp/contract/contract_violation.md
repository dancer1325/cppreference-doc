class contract_violation; |  |  (since C++26)  
| |   
  
The class `std::contracts::contract_violation` defines the type of objects used to represent a contract violation that has been detected during the evaluation of a [contract assertion](../language/contracts.html "cpp/language/contracts") with a particular evaluation semantic. 

Objects of this type can only be created by the implementation when the [contract-violation handler](../language/contracts.html#Contract-violation_handler "cpp/language/contracts") needs to be invoked. Users cannot create such objects directly.

##  std::contracts::contract_violation::evaluation_exception

[std::exception_ptr](../error/exception_ptr.html) evaluation_exception() const noexcept; |  |  (since C++26)  
---|---|---  
| |   
  
###  Return value

If the contract violation occurred because the evaluation of the predicate exited via an exception, returns an [std::exception_ptr](../error/exception_ptr.html "cpp/error/exception ptr") object that refers to that exception or a copy of that exception. Otherwise, returns a null [std::exception_ptr](../error/exception_ptr.html "cpp/error/exception ptr") object. 

##  std::contracts::contract_violation::comment

const char* comment() const noexcept; |  |  (since C++26)  
---|---|---  
| |   
  
###  Return value

Returns an implementation-defined [null-terminated multibyte string](../string/multibyte.html "cpp/string/multibyte") in the [ordinary literal encoding](../language/charset.html#Code_unit_and_literal_encoding "cpp/language/charset"). 

###  Notes

The C++ standard recommends that the string returned should contain a textual representation of the predicate of the violated contract assertion or an empty string if storing a textual representation is undesired. 

##  std::contracts::contract_violation::location

[std::source_location](../utility/source_location.html) location() const noexcept; |  |  (since C++26)  
---|---|---  
| |   
  
###  Return value

Returns a [std::source_location](../utility/source_location.html "cpp/utility/source location") object with implementation-defined value. 

###  Notes

The C++ standard recommends that the value returned should be a default constructed [std::source_location](../utility/source_location.html "cpp/utility/source location") object or a value identifying the violated contract assertion: 

  * When possible, if the violated contract assertion was a [precondition assertion](../language/function.html#Precondition_assertions "cpp/language/function"), the source location of the function invocation should be returned. 
  * Otherwise, the source location of the contract assertion should be returned.
