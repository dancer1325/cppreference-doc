`std::max_align_t` is a [standard-layout](../named_req/StandardLayoutType.html "cpp/named req/StandardLayoutType") [TrivialType](../named_req/TrivialType.html "cpp/named req/TrivialType")(until C++26)[TriviallyCopyable](../named_req/TriviallyCopyable.html "cpp/named req/TriviallyCopyable") type(since C++26) whose [alignment requirement](../language/objects.html#Alignment "cpp/language/object") is at least as strict (as large) as that of every scalar type. 

[std::is_trivially_default_constructible_v](is_default_constructible.html)<std::max_align_t> is true.

### [[edit](https://en.cppreference.com/mwiki/index.php?title=cpp/types/max_align_t&action=edit&section=1 "Edit section: Notes")] Notes

Pointers returned by allocation functions such as [std::malloc](../memory/c/malloc.html "cpp/memory/c/malloc") are suitably aligned for any object, which means they are aligned at least as strictly as `std::max_align_t`. 

### [[edit](https://en.cppreference.com/mwiki/index.php?title=cpp/types/max_align_t&action=edit&section=2 "Edit section: Example")] Example

Run this code
    
    
    #include <cstddef>
    #include <iostream>
    Â 
    int main()
    {
        [std::cout](../io/cout.html) << alignof(std::max_align_t) << '\n';
    }

Possible output: 
    
    
    16
