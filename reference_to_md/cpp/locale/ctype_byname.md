### Specializations

The standard library is guaranteed to provide the following specializations: 

Defined in header `[<locale>](../header/locale.html "cpp/header/locale")`  
---  
std::ctype_byname<char> |  provides narrow character classification (uses table lookup for character classification)   
std::ctype_byname<wchar_t> |  provides wide character classification

| |   
  
Constructs a new `std::ctype_byname` facet for a locale with name. 

refs is used for resource management: if refs == 0, the implementation destroys the facet, when the last [std::locale](locale.html "cpp/locale/locale") object holding it is destroyed. Otherwise, the object is not destroyed. 

###  Parameters

### Notes

std::ctype_byname<char> was incorrectly declared as an explicit specialization in the synopsis of [`<locale>`](../header/locale.html "cpp/header/locale"), and the declaration was removed by the resolution of [LWG issue 1298](https://cplusplus.github.io/LWG/issue1298), but it remains a required specialization, just like std::ctype_byname<wchar_t>. 

### Example

Run this code
    
    
    #include <iostream>
    #include <locale>
     
    int main()
    {
        wchar_t c = L'\u00de'; // capital letter thorn
     
        [std::locale](locale.html) loc("C");
     
        [std::cout](../io/cout.html) << "isupper('Þ', C locale) returned "
                  << [std::boolalpha](../io/manip/boolalpha.html) << [std::isupper](../string/byte/isupper.html)(c, loc) << '\n';
     
        loc = [std::locale](locale.html)(loc, new std::ctype_byname<wchar_t>("en_US.utf8"));
     
        [std::cout](../io/cout.html) << "isupper('Þ', C locale with Unicode ctype) returned "
                  << [std::boolalpha](../io/manip/boolalpha.html) << [std::isupper](../string/byte/isupper.html)(c, loc) << '\n';
    }

Output: 
    
    
    isupper('Þ', C locale) returned false
    isupper('Þ', C locale with Unicode ctype) returned true

### Defect reports

The following behavior-changing defect reports were applied retroactively to previously published C++ standards. 

DR  | Applied to  | Behavior as published  | Correct behavior   
---|---|---|---  
[LWG 16](https://cplusplus.github.io/LWG/issue16) | C++98  | the definition of the explicit specialization std::ctype_byname<char>  
misspecified the name and parameter list of `do_narrow` | corrected   
[LWG 616](https://cplusplus.github.io/LWG/issue616) | C++98  | the typename disambiguator was missing in the definition of `mask` | added
