Class template `std::money_get` encapsulates the rules for parsing monetary values from character streams. The standard I/O manipulator [std::get_money](../io/manip/get_money.html "cpp/io/manip/get money") uses the `std::money_get` facet of the I/O stream's locale. 

![std-money get-inheritance.svg](https://upload.cppreference.com/mwiki/images/1/1b/std-money_get-inheritance.svg)

Inheritance diagram

If a `std::money_get` specialization is not guaranteed to be provided by the standard library (see below), the behaviors of its [get()](money_get/get.html "cpp/locale/money get/get") and [do_get()](money_get/get.html "cpp/locale/money get/get") are not guaranteed as specified.

### Specializations

The standard library is guaranteed to provide the following specializations (they are [required to be implemented by any locale object](locale.html "cpp/locale/locale")): 

Defined in header `[<locale>](../header/locale.html "cpp/header/locale")`  
---  
std::money_get<char> |  parses narrow string representations of monetary values   
std::money_get<wchar_t> |  parses wide string representations of monetary values   
  
In addition, the standard library is also guaranteed to provide every specialization that satisfies the following type requirements: 

  * `CharT` is one of 
    * char, 
    * wchar_t, and 
    * any other implementation-defined [character container type](../string.html "cpp/string") that meets the requirements for a character on which any of the [iostream components](../io.html "cpp/io") can be instantiated; and 
  * `InputIt` must meet the requirements of [LegacyInputIterator](../named_req/InputIterator.html "cpp/named req/InputIterator").

### Example

Run this code
    
    
    #include <iomanip>
    #include <iostream>
    #include <iterator>
    #include <locale>
    #include <sstream>
     
    int main()
    {
        [std::string](../string/basic_string.html) str = "$1.11 $2.22 $3.33";
        [std::cout](../io/cout.html) << [std::fixed](../io/manip/fixed.html) << [std::setprecision](../io/manip/setprecision.html)(2);
     
        [std::cout](../io/cout.html) << '\"' << str << "\" parsed with the I/O manipulator: ";
        [std::istringstream](../io/basic_istringstream.html) s1(str);
        s1.imbue([std::locale](locale.html)("en_US.UTF-8"));
     
        long double val;
        while (s1 >> [std::get_money](../io/manip/get_money.html)(val))
            [std::cout](../io/cout.html) << val / 100 << ' ';
        [std::cout](../io/cout.html) << '\n';
     
        str = "USD  1,234.56";
        [std::cout](../io/cout.html) << '\"' << str << "\" parsed with the facet directly: ";
        [std::istringstream](../io/basic_istringstream.html) s2(str);
        s2.imbue([std::locale](locale.html)("en_US.UTF-8"));
     
        auto& f = [std::use_facet](use_facet.html)<std::money_get<char>>(s2.getloc());
        [std::ios_base::iostate](../io/ios_base/iostate.html) err;
        [std::istreambuf_iterator](../iterator/istreambuf_iterator.html)<char> beg(s2), end;
        f.get(beg, end, true, s2, err, val);
     
        [std::cout](../io/cout.html) << val / 100 << '\n';
    }

Output: 
    
    
    "$1.11 $2.22 $3.33" parsed with the I/O manipulator: 1.11 2.22 3.33
    "USD  1,234.56" parsed with the facet directly: 1234.56

### Defect reports

The following behavior-changing defect reports were applied retroactively to previously published C++ standards. 

DR  | Applied to  | Behavior as published  | Correct behavior   
---|---|---|---  
[LWG 427](https://cplusplus.github.io/LWG/issue427) | C++98  | `money_get` was guaranteed to accept any `CharT` that  
meets the requirements for a character on which  
any of the iostream components can be instantiated  | only guarantees to accept char,  
wchar_t and other implementation-  
defined character types   
[LWG 2392](https://cplusplus.github.io/LWG/issue2392) | C++98  | only character type `CharT` could be  
guaranteed to be accepted by `money_get` | can guarantee to accept implementation-  
defined character container types
