Compute the associated types of an iterator. 

1) Computes the [value type](../iterator.html#Types_and_writability "cpp/iterator") of `T`. 

  * If [std::iterator_traits](iterator_traits.html)<[std::remove_cvref_t](../types/remove_cvref.html)<T>> is not specialized, then std::iter_value_t<T> is [std::indirectly_readable_traits](readable_traits.html)<[std::remove_cvref_t](../types/remove_cvref.html)<T>>::value_type. 
  * Otherwise, it is [std::iterator_traits](iterator_traits.html)<[std::remove_cvref_t](../types/remove_cvref.html)<T>>::value_type.



2) Computes the _reference type_ of `T`.

3) Computes the _const reference type_ of `T`.

4) Computes the [difference type](../iterator.html#Types_and_writability "cpp/iterator") of `T`. 

  * If [std::iterator_traits](iterator_traits.html)<[std::remove_cvref_t](../types/remove_cvref.html)<T>> is not specialized, then std::iter_difference_t<T> is [std::incrementable_traits](incrementable_traits.html)<[std::remove_cvref_t](../types/remove_cvref.html)<T>>::difference_type. 
  * Otherwise, it is [std::iterator_traits](iterator_traits.html)<[std::remove_cvref_t](../types/remove_cvref.html)<T>>::difference_type.



5) Computes the _rvalue reference type_ of `T`. The constraint on this alias template is satisfied if and only if the expression [ranges::iter_move](ranges/iter_move.html)([std::declval](../utility/declval.html)<T&>()) is valid and has a referenceable type.

6) Computes the _common reference type_ of `T`. This is the common reference type between its reference type and an lvalue reference to its value type.

7) The exposition-only concept `_dereferenceable_` is satisfied if and only if the expression *[std::declval](../utility/declval.html)<T&>() is valid and has a [referenceable type](../meta.html#Definitions "cpp/meta").

8) The exposition-only alias template `_indirect-value-t_` denotes the following: 

  * [std::invoke_result_t](../types/result_of.html)<Proj&, /*indirect-value-t*/<I>> if `T` is the same as std::[projected](http://en.cppreference.com/w/cpp/ranges-placeholder/iterator/projected)<I, Proj> for some types `I` and `Proj`. 
  * Otherwise, std::iter_value_t<T>&.



### [[edit](https://en.cppreference.com/mwiki/index.php?title=cpp/iterator/iter_t&action=edit&section=1 "Edit section: Defect reports")] Defect reports

The following behavior-changing defect reports were applied retroactively to previously published C++ standards. 

DR  | Applied to  | Behavior as published  | Correct behavior   
---|---|---|---  
[P2609R3](https://wg21.link/P2609R3) | C++20  | `std::iter_common_reference_t` was defined in terms  
of std::iter_value_t<T>& which incorrectly handled  
[std::projected](projected.html "cpp/iterator/projected") types that project into rvalue reference types  | defined in terms of  
/*indirect-value-t*/<T>  
to handle such cases
