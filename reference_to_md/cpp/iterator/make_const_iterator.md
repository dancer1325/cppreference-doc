`std::basic_const_iterator` is an iterator adaptor which behaves exactly like the underlying iterator (which must be at least an [LegacyInputIterator](../named_req/InputIterator.html "cpp/named req/InputIterator") or model [`input_iterator`](input_iterator.html "cpp/iterator/input iterator")), except that dereferencing converts the value returned by the underlying iterator as immutable. Specializations of `std::basic_const_iterator` are constant iterators, that is, the iterator can never be used as an output iterator because modifying elements is not allowed.

### [[edit](https://en.cppreference.com/mwiki/index.php?title=cpp/iterator/basic_const_iterator&action=edit&section=9 "Edit section: Example")] Example

Run this code
    
    
    #include <cassert>
    #include <iterator>
    #include <vector>
     
    int main()
    {
        [std::vector](../container/vector.html) v{1, 2, 3};
        [std::vector](../container/vector.html)<int>::iterator i = v.begin();
        *i = 4;   // OK, v[0] == 4 now
        i[1] = 4; // OK, the same as *(i + 1) = 4;
     
        auto ci = [std::make_const_iterator](make_const_sentinel.html)(i);
        [assert](../error/assert.html)(*ci == 4);   // OK, can read the underlying object
        [assert](../error/assert.html)(ci[0] == 4); // OK, ditto
        // *ci = 13;        // Error: location is read-only
        // ci[0] = 13;      // Error: ditto
        ci.base()[0] = 42;  // OK, underlying iterator is writable
        [assert](../error/assert.html)(*ci == 42);  // OK, underlying location v[0] was modified
    }
