template< InputRange R, class Proj = [ranges::identity](../functional/identity.html),  
IndirectUnaryInvocable<[projected](http://en.cppreference.com/w/cpp/ranges-placeholder/iterator/projected)<[ranges::iterator_t](../../../ranges/iterator_t.html)<R>, Proj>> Fun >  
[ranges::tagged_pair](../utility/tagged_pair.html)<[tag::in](tags.html)([ranges::safe_iterator_t](../iterator/dangling.html)<R>), [tag::fun](tags.html)(Fun)>  
for_each( R&& r, Fun f, Proj proj = Proj{} ); |  (2)  |  (ranges TS)  
| |   
  
1) Invokes the given function object f to the result of invoking the projection proj on dereferencing every iterator in the range `[`first`, `last`)` (i.e.,[ranges::invoke](../functional/invoke.html)(f, [ranges::invoke](../functional/invoke.html)(proj, *i))), in order. 

2) Same as (1), but uses r as the source range, as if using [ranges::begin](../../../ranges/begin.html)(r) as first and [ranges::end](../../../ranges/end.html)(r) as last.

For both overloads, if the iterator type is mutable, f may modify the elements of the range through the dereferenced iterator. If f returns a result, the result is ignored. 

Unlike the rest of the algorithms, for_each is not allowed to make copies of the elements in the sequence even if they are trivially copyable. 

Unlike [std::for_each](../../../algorithm/for_each.html "cpp/algorithm/for each") (which requires only [MoveConstructible](../../../named_req/MoveConstructible.html "cpp/named req/MoveConstructible")), these functions require `Fun` to model [`CopyConstructible`](../concepts/CopyConstructible.html "cpp/experimental/ranges/concepts/CopyConstructible"). 

Notwithstanding the declarations depicted above, the actual number and order of template parameters for algorithm declarations is unspecified. Thus, if explicit template arguments are used when calling an algorithm, the program is probably non-portable.

first, last  |  \-  |  the range to apply the function to   
---|---|---  
r  |  \-  |  the range to apply the function to   
f  |  \-  |  callable object to be applied to each projected element in the range   
proj  |  \-  |  projection to apply to the elements   
  
### [[edit](https://en.cppreference.com/mwiki/index.php?title=cpp/experimental/ranges/algorithm/for_each&action=edit&section=2 "Edit section: Return value")] Return value

A `tagged_pair` object containing the following two members: 

  * The first member, with the tag `tag::in`, is the past-the-end iterator of the source range (that is, an iterator of type `I` that compares equal to the sentinel last). 
  * The second member, with the tag `tag::fun`, is initialized from `std::move(f)` (after all applications of the function object). 



### [[edit](https://en.cppreference.com/mwiki/index.php?title=cpp/experimental/ranges/algorithm/for_each&action=edit&section=3 "Edit section: Complexity")] Complexity

Exactly last \- first applications of f and proj. 

### [[edit](https://en.cppreference.com/mwiki/index.php?title=cpp/experimental/ranges/algorithm/for_each&action=edit&section=4 "Edit section: Possible implementation")] Possible implementation
    
    
    template<InputIterator I, Sentinel<I> S, class Proj = [ranges::identity](../functional/identity.html),
             IndirectUnaryInvocable<ranges::[projected](http://en.cppreference.com/w/cpp/ranges-placeholder/iterator/projected)<I, Proj>> Fun>
    auto for_each(I first, S last, Fun f, Proj proj = Proj{}) 
        -> [ranges::tagged_pair](../utility/tagged_pair.html)<[tag::in](tags.html)(I), [tag::fun](tags.html)(Fun)>
    {
        for (; first != last; ++first)
            [ranges::invoke](../functional/invoke.html)(f, [ranges::invoke](../functional/invoke.html)(proj, *first));
        return {std::move(first), std::move(f)};
    }
