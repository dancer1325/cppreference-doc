The standard comparison function objects are partially specialized for [std::experimental::propagate_const](../propagate_const.html)<T>. 

Let p.t_ denote the pointer-like object wrapped by a [std::experimental::propagate_const](../propagate_const.html)<T> p, then given objects `p` and `q` of type [std::experimental::propagate_const](../propagate_const.html)<T>, the following shall hold: 

  * [std::equal_to](../../utility/functional/equal_to.html)<[std::experimental::propagate_const](../propagate_const.html)<T>>()(p, q) == [std::equal_to](../../utility/functional/equal_to.html)<T>()(p.t_, q.t_)
  * [std::not_equal_to](../../utility/functional/not_equal_to.html)<[std::experimental::propagate_const](../propagate_const.html)<T>>()(p, q) == [std::not_equal_to](../../utility/functional/not_equal_to.html)<T>()(p.t_, q.t_)
  * [std::less](../../utility/functional/less.html)<[std::experimental::propagate_const](../propagate_const.html)<T>>()(p, q) == [std::less](../../utility/functional/less.html)<T>()(p.t_, q.t_)
  * [std::greater](../../utility/functional/greater.html)<[std::experimental::propagate_const](../propagate_const.html)<T>>()(p, q) == [std::greater](../../utility/functional/greater.html)<T>()(p.t_, q.t_)
  * [std::less_equal](../../utility/functional/less_equal.html)<[std::experimental::propagate_const](../propagate_const.html)<T>>()(p, q) == [std::less_equal](../../utility/functional/less_equal.html)<T>()(p.t_, q.t_)
  * [std::greater_equal](../../utility/functional/greater_equal.html)<[std::experimental::propagate_const](../propagate_const.html)<T>>()(p, q) == [std::greater_equal](../../utility/functional/greater_equal.html)<T>()(p.t_, q.t_)



### [[edit](https://en.cppreference.com/mwiki/index.php?title=cpp/experimental/propagate_const/cmp_func&action=edit&section=1 "Edit section: Notes")] Notes

These specializations ensure that when `T` is a pointer type, specializations of these class templates for [std::experimental::propagate_const](../propagate_const.html)<T> yield a total order, even if the corresponding built-in operators do not.
