void resize( size_type count ); |  (1) | (constexpr since C++26)  
---|---|---  
void resize( size_type count, const value_type& value ); |  (2) | (constexpr since C++26)  
| |   
  
Resizes the container to contain count elements: 

  * If count is equal to the current size, does nothing. 
  * If the current size is greater than count, the container is reduced to its first count elements. 
  * If the current size is less than count, then: 



1) Additional copies of T()(until C++11)[default-inserted](../../named_req/DefaultInsertable.html "cpp/named req/DefaultInsertable") elements(since C++11) are appended.

2) Additional copies of value are appended.

count  |  \-  |  new size of the container   
---|---|---  
value  |  \-  |  the value to initialize the new elements with   
Type requirements   
- |  If the following condition is satisfied, the behavior is undefined:  1) `T` is not [DefaultInsertable](../../named_req/DefaultInsertable.html "cpp/named req/DefaultInsertable") or [MoveInsertable](../../named_req/MoveInsertable.html "cpp/named req/MoveInsertable") into `deque`. 2) `T` is not [CopyInsertable](../../named_req/CopyInsertable.html "cpp/named req/CopyInsertable") into `deque`. | (since C++11)  
---|---  
  
### [[edit](https://en.cppreference.com/mwiki/index.php?title=Template:cpp/container/resize&action=edit&section=T-2 "Template:cpp/container/resize")] Complexity

Linear in the difference between the current size and count. 

###  Notes

If value-initialization in overload ([1](resize.html#Version_1)) is undesirable, for example, if the elements are of non-class type and zeroing out is not needed, it can be avoided by providing a [custom `Allocator::construct`](https://stackoverflow.com/a/21028912/273767). 

### [[edit](https://en.cppreference.com/mwiki/index.php?title=Template:cpp/container/resize&action=edit&section=T-6 "Template:cpp/container/resize")] Example

Run this code
    
    
    #include <deque>
    #include <iostream>
     
    void print(auto rem, const [std::deque](../deque.html)<int>& c)
    {
        for ([std::cout](../../io/cout.html) << rem; const int el : c)
            [std::cout](../../io/cout.html) << el << ' ';
        [std::cout](../../io/cout.html) << '\n';
    }
     
    int main()
    {
        [std::deque](../deque.html)<int> c = {1, 2, 3};
        print("The deque holds: ", c);
     
        c.resize(5);
        print("After resize up to 5: ", c);
     
        c.resize(2);
        print("After resize down to 2: ", c);
     
        c.resize(6, 4);
        print("After resize up to 6 (initializer = 4): ", c);
    }

Output: 
    
    
    The deque holds: 1 2 3
    After resize up to 5: 1 2 3 0 0
    After resize down to 2: 1 2
    After resize up to 6 (initializer = 4): 1 2 4 4 4 4

###  Defect reports

The following behavior-changing defect reports were applied retroactively to previously published C++ standards. 

DR  | Applied to  | Behavior as published  | Correct behavior   
---|---|---|---  
[LWG 679](https://cplusplus.github.io/LWG/issue679) | C++98  | `resize()` passed value by value  | passes by const reference   
[LWG 1418](https://cplusplus.github.io/LWG/issue1418) | C++98  | the behavior of resize(size()) was not specified  | specified   
[LWG 2033](https://cplusplus.github.io/LWG/issue2033) | C++11  | 1\. elements were removed by using [erase()](erase.html "cpp/container/deque/erase")[[1]](resize.html#cite_note-1)  
2\. `T` was not required to be [MoveInsertable](../../named_req/MoveInsertable.html "cpp/named req/MoveInsertable") | 1\. uses [pop_back()](pop_back.html "cpp/container/deque/pop back")  
2\. required   
[LWG 2066](https://cplusplus.github.io/LWG/issue2066) | C++11  | overload ([1](resize.html#Version_1)) did not have the exception  
safety guarantee of overload ([2](resize.html#Version_2)) | added   
[LWG 2160](https://cplusplus.github.io/LWG/issue2160) | C++11  | elements were removed by using [pop_back()](pop_back.html "cpp/container/deque/pop back")[[2]](resize.html#cite_note-2)  
due to the resolution of LWG 2033  | does not specify the method  
of removing elements   
  
  1. [↑](resize.html#cite_ref-1) [erase()](erase.html "cpp/container/deque/erase") can remove elements in the middle of a `deque`, so the value type is required to be [MoveAssignable](../../named_req/MoveAssignable.html "cpp/named req/MoveAssignable") so that the elements following the removed section can be moved forward to fill the gap. However, `resize()` can only remove elements at the end of the `deque`, making the [MoveAssignable](../../named_req/MoveAssignable.html "cpp/named req/MoveAssignable") unnecessary.
  2. [↑](resize.html#cite_ref-2) Removing elements by using [pop_back()](pop_back.html "cpp/container/deque/pop back") implies that the elements are required to be removed from back to front.
