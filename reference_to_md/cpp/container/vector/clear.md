void clear(); |  | (noexcept since C++11)   
(constexpr since C++20)  
---|---|---  
| |   
  
Erases all elements from the container. After this call, [size()](size.html "cpp/container/vector/size") returns zero. 

Invalidates any references, pointers, and iterators referring to contained elements. Any past-the-end iterators are also invalidated.

### [[edit](https://en.cppreference.com/mwiki/index.php?title=Template:cpp/container/clear&action=edit&section=T-1 "Template:cpp/container/clear")] Complexity

Linear in the size of the container, i.e., the number of elements. 

###  Notes

Calling `clear()` does not affect the result of [capacity()](capacity.html "cpp/container/vector/capacity"). The standard's restriction on the changes to capacity is in the specification of [reserve()](reserve.html "cpp/container/vector/reserve"), see [SO](https://stackoverflow.com/a/18467916). 

### [[edit](https://en.cppreference.com/mwiki/index.php?title=Template:cpp/container/clear&action=edit&section=T-3 "Template:cpp/container/clear")] Example

Run this code
    
    
    #include <iostream>
    #include <string_view>
    #include <vector>
     
    void print_info([std::string_view](../../string/basic_string_view.html) rem, const [std::vector](../vector.html)<int>& v)
    {
        [std::cout](../../io/cout.html) << rem << "{ ";
        for (const auto& value : v)
            [std::cout](../../io/cout.html) << value << ' ';
        [std::cout](../../io/cout.html) << "}\n";
        [std::cout](../../io/cout.html) << "Size=" << v.size() << ", Capacity=" << v.capacity() << '\n';
    }
     
    int main()
    {
        [std::vector](../vector.html)<int> container{1, 2, 3};
        print_info("Before clear: ", container);
        container.clear();
        print_info("After clear: ", container);
    }

Output: 
    
    
    Before clear: { 1 2 3 }
    Size=3, Capacity=3
    After clear: { }
    Size=0, Capacity=3

###  Defect reports

The following behavior-changing defect reports were applied retroactively to previously published C++ standards. 

DR  | Applied to  | Behavior as published  | Correct behavior   
---|---|---|---  
[LWG 2231](https://cplusplus.github.io/LWG/issue2231) | C++11  | complexity guarantee was mistakenly omitted in C++11  | complexity reaffirmed as linear
