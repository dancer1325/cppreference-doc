The [`std::vector`](../vector_bool.html "cpp/container/vector bool")<bool, Alloc> specializations define [`std::vector`](../vector_bool.html "cpp/container/vector bool")<bool, Alloc>::reference as a publicly-accessible nested class. [`std::vector`](../vector_bool.html "cpp/container/vector bool")<bool, Alloc>::reference proxies the behavior of references to a single bit in [`std::vector`](../vector_bool.html "cpp/container/vector bool")<bool, Alloc>. 

The primary use of [`std::vector`](../vector_bool.html "cpp/container/vector bool")<bool, Alloc>::reference is to provide an assignable value that can be returned from operator[]. 

Any reads or writes to a vector that happen via a [`std::vector`](../vector_bool.html "cpp/container/vector bool")<bool, Alloc>::reference potentially read or write to the entire underlying vector.

##  std::formatter<std::vector<bool, Alloc>::reference>

template< class T, class CharT >  
requires /*is-vector-bool-reference*/<T>  
struct formatter<T, CharT>; |  |  (since C++23)  
---|---|---  
| |   
  
Specializes the [std::formatter](../../utility/format/formatter.html "cpp/utility/format/formatter") for [`std::vector`](../vector_bool.html "cpp/container/vector bool")<bool, Alloc>::reference. The specialization uses [std::formatter](../../utility/format/formatter.html)<bool, CharT> as its underlying formatter (denoted as `_underlying__`) where the referenced bit is converted to bool to be formatted. 

The exposition-only constant /*is-vector-bool-reference*/<T> is true if and only if `T` denotes the type [`std::vector`](../vector_bool.html "cpp/container/vector bool")<bool, Alloc>::reference for some type `Alloc` and [`std::vector`](../vector_bool.html "cpp/container/vector bool")<bool, Alloc> is not a [program-defined specialization](../../language/type-id.html#Program-defined_type "cpp/language/type").
