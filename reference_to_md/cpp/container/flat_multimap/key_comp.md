key_compare key_comp() const; |  | (since C++23)   
(constexpr since C++26)  
---|---|---  
| |   
  
Returns the function object that compares the keys, which is a copy of the key comparison object used by *this.

### [[edit](https://en.cppreference.com/mwiki/index.php?title=Template:cpp/container/key_comp&action=edit&section=T-1 "Template:cpp/container/key comp")] Return value

The key comparison function object. 

### [[edit](https://en.cppreference.com/mwiki/index.php?title=Template:cpp/container/key_comp&action=edit&section=T-2 "Template:cpp/container/key comp")] Complexity

Constant. 

### [[edit](https://en.cppreference.com/mwiki/index.php?title=Template:cpp/container/key_comp&action=edit&section=T-3 "Template:cpp/container/key comp")] Example

Run this code
    
    
    #include <iostream>
    #include <flat_map>
    #include <utility>
     
    // Example module 97 key compare function
    struct ModCmp
    {
        bool operator()(int lhs, int rhs) const
        {
            return (lhs % 97) < (rhs % 97);
        }
    };
     
    int main()
    {
        [std::flat_multimap](../flat_multimap.html)<int, char, ModCmp> cont;
        cont = {{1, 'a'}, {2, 'b'}, {3, 'c'}, {4, 'd'}, {5, 'e'}};
     
        auto comp_func = cont.key_comp();
     
        for (const auto it : cont)
        {
            const bool before = comp_func(it.first, 100);
            const bool after = comp_func(100, it.first);
     
            [std::cout](../../io/cout.html) << "Key (" << it.first << ',' << it.second << ") ";
            if (!before && !after)
                [std::cout](../../io/cout.html) << "equivalent to key (100)\n";
            else if (before)
                [std::cout](../../io/cout.html) << "goes before key (100)\n";
            else if (after)
                [std::cout](../../io/cout.html) << "goes after key (100)\n";
            else
                [std::unreachable](../../utility/unreachable.html)();
        }
    }

Output: 
    
    
    (1,a) goes before key (100)
    (2,b) goes before key (100)
    (3,c) equivalent to key (100)
    (4,d) goes after key (100)
    (5,e) goes after key (100)
