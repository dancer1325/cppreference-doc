Constructs new container adaptor from a variety of data sources and optionally provided comparison function object comp and/or allocator alloc. 

1) A default constructor. Constructs an empty container adaptor.

2) A [copy constructor](../../language/copy_constructor.html "cpp/language/copy constructor"). Constructs [`_c_`](../flat_set.html#Member_objects "cpp/container/flat set") with the copy of the contents of other.c and [`_compare_`](../flat_set.html#Member_objects "cpp/container/flat set") with other.compare. See [allocator usage note](flat_set.html#Allocator_usage_note) below.

3) A [move constructor](../../language/move_constructor.html "cpp/language/move constructor"). Constructs the container adaptor with the contents of other using move semantics. See [allocator usage note](flat_set.html#Allocator_usage_note) below.

4) Constructs the underlying container with the contents of the container cont. First, initializes [`_c_`](../flat_set.html#Member_objects "cpp/container/flat set") with std::move(cont) and [`_compare_`](../flat_set.html#Member_objects "cpp/container/flat set") with comp. Then sorts the [`_c_`](../flat_set.html#Member_objects "cpp/container/flat set") with respect to `comp`. Finally, makes elements unique, i.e. erases all but the first element from each group of consecutive equivalent elements.

5) Same as (4), equivalent to flat_set(cont);. See [allocator usage note](flat_set.html#Allocator_usage_note) below.

6) Same as (4), equivalent to flat_set(cont, comp);. See [allocator usage note](flat_set.html#Allocator_usage_note) below.

7) Constructs the underlying container with the contents of the other container cont. Initializes [`_c_`](../flat_set.html#Member_objects "cpp/container/flat set") with std::move(cont) and [`_compare_`](../flat_set.html#Member_objects "cpp/container/flat set") with comp.

8) Same as (7), equivalent to flat_set(s, cont);. See [allocator usage note](flat_set.html#Allocator_usage_note) below.

9) Same as (7), equivalent to flat_set(s, cont, comp);. See [allocator usage note](flat_set.html#Allocator_usage_note) below.

10) Constructs an empty container adaptor.

11,12) Constructs an empty container adaptor. See [allocator usage note](flat_set.html#Allocator_usage_note) below.

13) Constructs the container adaptor with the contents of the range `[`first`, `last`)`, equivalent to insert(first, last);.

14,15) Same as (13). See [allocator usage note](flat_set.html#Allocator_usage_note) below.

16) Constructs the container adaptor with the contents of the range rg. First, uses (10) as [delegating constructor](../../language/initializer_list.html#Delegating_constructor "cpp/language/constructor"). Then initializes [`_c_`](../flat_set.html#Member_objects "cpp/container/flat set") with the contents of rg as if by insert_range([std::forward](../../utility/forward.html)<R>(rg));.

17) Same as (16) using it as [delegating constructor](../../language/initializer_list.html#Delegating_constructor "cpp/language/constructor").

18,19) Same as (16). See [allocator usage note](flat_set.html#Allocator_usage_note) below.

20) Constructs the underlying container with the contents of the range `[`first`, `last`)`. Initializes [`_c_`](../flat_set.html#Member_objects "cpp/container/flat set") with c(first, last) and [`_compare_`](../flat_set.html#Member_objects "cpp/container/flat set") with compare(comp).

21,22) Same as (20). See [allocator usage note](flat_set.html#Allocator_usage_note) below.

23) An [initializer-list constructor](../../language/list_initialization.html "cpp/language/list initialization"). Constructs the underlying container with the contents of the initializer list init, using (13) as [delegating constructor](../../language/initializer_list.html#Delegating_constructor "cpp/language/constructor").

24,25) Same as (23). See [allocator usage note](flat_set.html#Allocator_usage_note) below.

26) An [initializer-list constructor](../../language/list_initialization.html "cpp/language/list initialization"). Constructs the underlying container with the contents of the initializer list init, using (20) as [delegating constructor](../../language/initializer_list.html#Delegating_constructor "cpp/language/constructor").

27,28) Save as (26). See [allocator usage note](flat_set.html#Allocator_usage_note) below.

Note for overloads (13-15,20-22): If `[`first`, `last`)` is not a [valid range](../../iterator.html#Ranges "cpp/iterator"), the behavior is undefined. 

Note for overloads (4-6,13-19,23-25): If multiple elements in the range have keys that compare equivalent, it is unspecified which element is inserted (pending [LWG2844](https://cplusplus.github.io/LWG/issue2844)).

##### Allocator usage note

The constructors (2,3,5,6,8,9,11,12,14,15,17,19,21,22,24,25,27,28) are equivalent to the corresponding non-allocator constructors except that [`_c_`](../flat_set.html#Member_objects "cpp/container/flat set") is constructed with [uses-allocator construction](../../memory/uses_allocator.html#Uses-allocator_construction "cpp/memory/uses allocator"). These overloads participate in overload resolution only if [std::uses_allocator_v](../../memory/uses_allocator.html)<container_type, Allocator> is true. 

### Parameters

cont  |  \-  |  a container to be used as source to initialize the underlying container   
---|---|---  
other  |  \-  |  another `flat_set` to be used as source to initialize the elements of the underlying container with   
alloc  |  \-  |  an allocator to use for all memory allocations of the underlying container   
comp  |  \-  |  a function object to be used for all comparisons of keys   
first, last  |  \-  |  the pair of iterators defining the source [range](../../iterator.html#Ranges "cpp/iterator") of elements to copy   
init  |  \-  |  an initializer list to initialize the elements of the underlying container with   
rg  |  \-  |  a [container compatible range](../../ranges/to.html#container_compatible_range "cpp/ranges/to") (that is, an [`input_range`](../../ranges/input_range.html "cpp/ranges/input range") whose elements are convertible to [`value_type`](../flat_set.html#Member_types "cpp/container/flat set")) to be used as source to initialize the underlying container   
fr  |  \-  |  a [disambiguation tag](../../ranges/from_range.html "cpp/ranges/from range") that indicates that the contained member should be range constructed   
s  |  \-  |  a [disambiguation tag](../sorted_unique.html "cpp/container/sorted unique") that indicates that the input sequence is sorted with respect to [`_compare_`](../flat_set.html#Member_objects "cpp/container/flat set") and all its elements are unique   
Type requirements   
-`InputIt` must meet the requirements of [LegacyInputIterator](../../named_req/InputIterator.html "cpp/named req/InputIterator").   
-`Compare` must meet the requirements of [Compare](../../named_req/Compare.html "cpp/named req/Compare").   
-`Allocator` must meet the requirements of [Allocator](../../named_req/Allocator.html "cpp/named req/Allocator").   
  
### Complexity

1) Constant.

2) Linear in size of other.

3) Same as the corresponding move-constructor of the wrapped container, i.e. constant or linear in size of cont.

4-6) Linear in \\(\scriptsize N\\)N if cont is sorted with respect to compare, otherwise \\(\scriptsize \mathcal{O}(N\cdot\log{(N)})\\)(N路log(N)), where \\(\scriptsize N\\)N is the value of cont.size() before this call.

7-9) Same as the corresponding move-constructor of the wrapped container, i.e. constant or linear in size of cont.

10-12) Constant.

13-15) Linear in \\(\scriptsize N\\)N if the input range `[`first`, `last`)` is sorted with respect to compare, otherwise \\(\scriptsize \mathcal{O}(N\cdot\log{(N)})\\)(N路log(N)), where \\(\scriptsize N\\)N is the value of cont.size() before this call.

16-19) Linear in \\(\scriptsize N\\)N if the input range rg is sorted with respect to compare, otherwise \\(\scriptsize \mathcal{O}(N\cdot\log{(N)})\\)(N路log(N)), where \\(\scriptsize N\\)N is the value of cont.size() before this call.

20-22) Linear in size of `[`first`, `last`)`.

23-25) Linear in \\(\scriptsize N\\)N if the elements of init are sorted with respect to compare, otherwise \\(\scriptsize \mathcal{O}(N\cdot\log{(N)})\\)(N路log(N)), where \\(\scriptsize N\\)N is the value of cont.size() before this call.

26-28) Linear in size of init.

### Exceptions

Calls to `Allocator::allocate` may throw. 

### Notes

After container move construction (overload ([3](flat_set.html#Version_3))), references, pointers, and iterators (other than the end iterator) to other remain valid, but refer to elements that are now in *this. The current standard makes this guarantee via the blanket statement in [[container.reqmts]/67](https://eel.is/c++draft/container.reqmts#67), and a more direct guarantee is under consideration via [LWG issue 2321](https://cplusplus.github.io/LWG/issue2321).
