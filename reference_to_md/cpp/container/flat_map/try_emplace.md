If a key equivalent to k already exists in the container, does nothing. Otherwise, inserts a new element into the underlying containers [`_c_`](../flat_map.html#Member_objects "cpp/container/flat map") with key k and value constructed with args. 

1,2,4,5) Equivalent to: 
    
    
    auto key_it = [ranges::upper_bound](../../algorithm/ranges/upper_bound.html)(c.keys, k, compare);
    auto value_it = c.values.begin() + [std::distance](../../iterator/distance.html)(c.keys.begin(), key_it);
    c.keys.insert(key_it, [std::forward](../../utility/forward.html)<decltype(k)>(k));
    c.values.emplace(value_it, [std::forward](../../utility/forward.html)<Args>(args)...);

3,6) Equivalent to: 
    
    
    auto key_it = [ranges::upper_bound](../../algorithm/ranges/upper_bound.html)(c.keys, k, compare);
    auto value_it = c.values.begin() + [std::distance](../../iterator/distance.html)(c.keys.begin(), key_it);
    c.keys.emplace(key_it, [std::forward](../../utility/forward.html)<K>(k));
    c.values.emplace(value_it, [std::forward](../../utility/forward.html)<Args>(args)...);

The conversion from k into `key_type` must construct an object u, for which find(k) == find(u) is true. Otherwise, the behavior is undefined.

These overloads participate in overload resolution only if : 

  * The qualified-id `Compare::is_transparent` is valid and denotes a type. 
  * [std::is_constructible_v](../../types/is_constructible.html)<key_type, K> is true. 
  * [std::is_assignable_v](../../types/is_assignable.html)<mapped_type&, Args...> is true. 
  * For (3) only, [std::is_convertible_v](../../types/is_convertible.html)<K&&, const_iterator> and [std::is_convertible_v](../../types/is_convertible.html)<K&&, iterator> are both false.

### Parameters

k  |  \-  |  the key used both to look up and to insert if not found   
---|---|---  
hint  |  \-  |  iterator to the position before which the new element will be inserted   
args  |  \-  |  arguments to forward to the constructor of the element   
  
### Return value

1-3) Same as for [`emplace`](emplace.html "cpp/container/flat map/emplace").

4-6) Same as for [`emplace_hint`](emplace_hint.html "cpp/container/flat map/emplace hint").

### Complexity

1-3) Same as for [`emplace`](emplace.html "cpp/container/flat map/emplace").

4-6) Same as for [`emplace_hint`](emplace_hint.html "cpp/container/flat map/emplace hint").

### Notes

Unlike [`insert`](insert.html "cpp/container/flat map/insert") or [`emplace`](emplace.html "cpp/container/flat map/emplace"), these functions do not move from rvalue arguments if the insertion does not happen, which makes it easy to manipulate maps whose values are move-only types, such as [std::flat_map](../flat_map.html)<[std::string](../../string/basic_string.html), [std::unique_ptr](../../memory/unique_ptr.html)<foo>>. In addition, `try_emplace` treats the key and the arguments to the `mapped_type` separately, unlike [`emplace`](emplace.html "cpp/container/flat map/emplace"), which requires the arguments to construct a `value_type` (that is, a [std::pair](../../utility/pair.html "cpp/utility/pair")). 

Overloads ([3,6](try_emplace.html#Version_3)) can be called without constructing an object of type `key_type`. 

### Example

Run this code
    
    
    #include <flat_map>
    #include <iostream>
    #include <string>
    #include <utility>
     
    void print_node(const auto& node)
    {
        [std::cout](../../io/cout.html) << '[' << node.first << "] = " << node.second << '\n';
    }
     
    void print_result(auto const& pair)
    {
        [std::cout](../../io/cout.html) << (pair.second ? "inserted: " : "ignored:  ");
        print_node(*pair.first);
    }
     
    int main()
    {
        using namespace std::literals;
        [std::map](../map.html)<[std::string](../../string/basic_string.html), [std::string](../../string/basic_string.html)> m;
     
        print_result(m.try_emplace( "a", "a"s));
        print_result(m.try_emplace( "b", "abcd"));
        print_result(m.try_emplace( "c", 10, 'c'));
        print_result(m.try_emplace( "c", "Won't be inserted"));
     
        for (const auto& p : m)
            print_node(p);
    }

Output: 
    
    
    inserted: [a] = a
    inserted: [b] = abcd
    inserted: [c] = cccccccccc
    ignored:  [c] = cccccccccc
    [a] = a
    [b] = abcd
    [c] = cccccccccc
