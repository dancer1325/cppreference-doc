If a key equivalent to k already exists in the container, does nothing. Otherwise, inserts a new element into the container with key k and value constructed with args. In such case: 

1) Behaves like [`emplace`](emplace.html "cpp/container/map/emplace") except that the element is constructed as  
value_type([std::piecewise_construct](../../utility/piecewise_construct.html),  


[std::forward_as_tuple](../../utility/tuple/forward_as_tuple.html)(k),  


[std::forward_as_tuple](../../utility/tuple/forward_as_tuple.html)([std::forward](../../utility/forward.html)<Args>(args)...))

2) Behaves like [`emplace`](emplace.html "cpp/container/map/emplace") except that the element is constructed as  
value_type([std::piecewise_construct](../../utility/piecewise_construct.html),  


[std::forward_as_tuple](../../utility/tuple/forward_as_tuple.html)(std::move(k)),  


[std::forward_as_tuple](../../utility/tuple/forward_as_tuple.html)([std::forward](../../utility/forward.html)<Args>(args)...))

3) Behaves like [`emplace`](emplace.html "cpp/container/map/emplace") except that the element is constructed as  
value_type([std::piecewise_construct](../../utility/piecewise_construct.html),  


[std::forward_as_tuple](../../utility/tuple/forward_as_tuple.html)([std::forward](../../utility/forward.html)<K>(k)),  


[std::forward_as_tuple](../../utility/tuple/forward_as_tuple.html)([std::forward](../../utility/forward.html)<Args>(args)...))

4) Behaves like [`emplace_hint`](emplace_hint.html "cpp/container/map/emplace hint") except that the element is constructed as  
value_type([std::piecewise_construct](../../utility/piecewise_construct.html),  


[std::forward_as_tuple](../../utility/tuple/forward_as_tuple.html)(k),  


[std::forward_as_tuple](../../utility/tuple/forward_as_tuple.html)([std::forward](../../utility/forward.html)<Args>(args)...))

5) Behaves like [`emplace_hint`](emplace_hint.html "cpp/container/map/emplace hint") except that the element is constructed as  
value_type([std::piecewise_construct](../../utility/piecewise_construct.html),  


[std::forward_as_tuple](../../utility/tuple/forward_as_tuple.html)(std::move(k)),  


[std::forward_as_tuple](../../utility/tuple/forward_as_tuple.html)([std::forward](../../utility/forward.html)<Args>(args)...))

6) Behaves like [`emplace_hint`](emplace_hint.html "cpp/container/map/emplace hint") except that the element is constructed as  
value_type([std::piecewise_construct](../../utility/piecewise_construct.html),  


[std::forward_as_tuple](../../utility/tuple/forward_as_tuple.html)([std::forward](../../utility/forward.html)<K>(k)),  


[std::forward_as_tuple](../../utility/tuple/forward_as_tuple.html)([std::forward](../../utility/forward.html)<Args>(args)...))

1-6) If `value_type` is not [EmplaceConstructible](../../named_req/EmplaceConstructible.html "cpp/named req/EmplaceConstructible") into `map` from the corresponding expression, the behavior is undefined.

3) This overload participates in overload resolution only if all following conditions are satisfied: 

  * [std::is_convertible_v](../../types/is_convertible.html)<K&&, const_iterator> and [std::is_convertible_v](../../types/is_convertible.html)<K&&, iterator> are both false. 
  * The qualified-id Compare::is_transparent is valid and denotes a type.



If equal_range(u.first) == equal_range(k) is false, the behavior is undefined, where u is the new element to be inserted.

6) This overload participates in overload resolution only if the qualified-id Compare::is_transparent is valid and denotes a type.

If equal_range(u.first) == equal_range(k) is false, the behavior is undefined, where u is the new element to be inserted.

No iterators or references are invalidated.

### [[edit](https://en.cppreference.com/mwiki/index.php?title=Template:cpp/container/try_emplace_assoc&action=edit&section=T-1 "Template:cpp/container/try emplace assoc")] Parameters

k  |  \-  |  the key used both to look up and to insert if not found   
---|---|---  
hint  |  \-  |  iterator to the position before which the new element will be inserted   
args  |  \-  |  arguments to forward to the constructor of the element   
  
### [[edit](https://en.cppreference.com/mwiki/index.php?title=Template:cpp/container/try_emplace_assoc&action=edit&section=T-2 "Template:cpp/container/try emplace assoc")] Return value

1-3) Same as for [`emplace`](emplace.html "cpp/container/map/emplace"):  
A pair consisting of an iterator to the inserted element (or to the element that prevented the insertion) and a bool value set to true if and only if the insertion took place.

4-6) Same as for [`emplace_hint`](emplace_hint.html "cpp/container/map/emplace hint"):  
An iterator to the inserted element, or to the element that prevented the insertion.

### [[edit](https://en.cppreference.com/mwiki/index.php?title=Template:cpp/container/try_emplace_assoc&action=edit&section=T-3 "Template:cpp/container/try emplace assoc")] Complexity

1-3) Same as for [`emplace`](emplace.html "cpp/container/map/emplace"):  
Logarithmic in the size of the container.

4-6) Same as for [`emplace_hint`](emplace_hint.html "cpp/container/map/emplace hint"):  
Logarithmic in the size of the container in general, but amortized constant if the new element is inserted just before hint.

### [[edit](https://en.cppreference.com/mwiki/index.php?title=Template:cpp/container/try_emplace_assoc&action=edit&section=T-4 "Template:cpp/container/try emplace assoc")] Notes

Unlike [`insert`](insert.html "cpp/container/map/insert") or [`emplace`](emplace.html "cpp/container/map/emplace"), these functions do not move from rvalue arguments if the insertion does not happen, which makes it easy to manipulate maps whose values are move-only types, such as [std::map](../map.html)<[std::string](../../string/basic_string.html), [std::unique_ptr](../../memory/unique_ptr.html)<foo>>. In addition, `try_emplace` treats the key and the arguments to the `mapped_type` separately, unlike [`emplace`](emplace.html "cpp/container/map/emplace"), which requires the arguments to construct a `value_type` (that is, a [std::pair](../../utility/pair.html "cpp/utility/pair")). 

Overloads ([3](try_emplace.html#Version_3)) and ([6](try_emplace.html#Version_6)) can be called without constructing an object of type `Key`.

### [[edit](https://en.cppreference.com/mwiki/index.php?title=Template:cpp/container/try_emplace_assoc&action=edit&section=T-5 "Template:cpp/container/try emplace assoc")] Example

Run this code
    
    
    #include <iostream>
    #include <string>
    #include <map>
    #include <utility>
     
    void print_node(const auto& node)
    {
        [std::cout](../../io/cout.html) << '[' << node.first << "] = " << node.second << '\n';
    }
     
    void print_result(auto const& pair)
    {
        [std::cout](../../io/cout.html) << (pair.second ? "inserted: " : "ignored:  ");
        print_node(*pair.first);
    }
     
    int main()
    {
        using namespace std::literals;
        [std::map](../map.html)<[std::string](../../string/basic_string.html), [std::string](../../string/basic_string.html)> m;
     
        print_result(m.try_emplace("a", "a"s));
        print_result(m.try_emplace("b", "abcd"));
        print_result(m.try_emplace("c", 10, 'c'));
        print_result(m.try_emplace("c", "Won't be inserted"));
     
        for (const auto& p : m)
            print_node(p);
    }

Output: 
    
    
    inserted: [a] = a
    inserted: [b] = abcd
    inserted: [c] = cccccccccc
    ignored:  [c] = cccccccccc
    [a] = a
    [b] = abcd
    [c] = cccccccccc
