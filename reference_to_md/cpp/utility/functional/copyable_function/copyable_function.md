Creates a new `std::copyable_function`. 

1,2) Default constructor and the constructor taking nullptr construct an empty `std::copyable_function`.

3) Copy constructor constructs a `std::copyable_function` whose target is a copy of the target of other. Otherwise, constructs an empty `std::copyable_function` if other is empty.

4) Move constructor constructs a `std::copyable_function` whose target is that of other. other is in a valid but unspecified state after move construction.

5) Let `VT` be [std::decay_t](../../../types/decay.html)<F>. If f is a null function pointer, a null pointer to member value, or an empty `std::copyable_function` (may be any other specialization), then constructs an empty `std::copyable_function`. Otherwise, constructs a `std::copyable_function` whose target is of type `VT` and direct-non-list-initialized with [std::forward](../../forward.html)<F>(f). 

  * This overload participates in overload resolution only if `VT` is neither same as `copyable_function` nor a specialization of [std::in_place_type_t](../../in_place.html "cpp/utility/in place"), and /*is-callable-from*/<VT> (see below) is true. 
  * The program is ill-formed if [std::is_constructible_v](../../../types/is_constructible.html)<VT, F> or [std::is_copy_constructible_v](../../../types/is_copy_constructible.html)<VT> is not true.



6) Let `VT` be [std::decay_t](../../../types/decay.html)<T>. Constructs a `std::copyable_function` whose target is of type `VT` and direct-non-list-initialized with [std::forward](../../forward.html)<CArgs>(args).... 

  * This overload participates in overload resolution only if both [std::is_constructible_v](../../../types/is_constructible.html)<VT, CArgs...> and /*is-callable-from*/<VT> (see below) are true. 
  * The program is ill-formed if `VT` is not the same type as `T` or [std::is_copy_constructible_v](../../../types/is_copy_constructible.html)<VT> is not true.



7) Let `VT` be [std::decay_t](../../../types/decay.html)<T>. Constructs a `std::copyable_function` whose target is of type `VT` and direct-non-list-initialized with il, [std::forward](../../forward.html)<CArgs>(args).... 

  * This overload participates in overload resolution only if both [std::is_constructible_v](../../../types/is_constructible.html)<VT, [std::initializer_list](../../initializer_list.html)<U>&, CArgs...> and /*is-callable-from*/<VT> (see below) are true. 
  * The program is ill-formed if `VT` is not the same type as `T` or [std::is_copy_constructible_v](../../../types/is_copy_constructible.html)<VT> is not true.



For constructors (5-7), the behavior is undefined unless `VT` satisfies both [Destructible](../../../named_req/Destructible.html "cpp/named req/Destructible") and [CopyConstructible](../../../named_req/CopyConstructible.html "cpp/named req/CopyConstructible") requirements.
