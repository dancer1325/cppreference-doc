The template specialization of [std::formatter](formatter.html "cpp/utility/format/formatter") for the range types allows users to convert a range to its textual representation as a collection of elements or a string using [formatting functions](../format.html "cpp/utility/format"). 

The specialization is derived from `_range-default-formatter_` <[std::format_kind](format_kind.html)<R>, R, CharT>. 

The specialization is enabled if `R` satisfies [`input_range`](../../ranges/input_range.html "cpp/ranges/input range"), [std::format_kind](format_kind.html)<R> is not std::range_format::disabled, and [std::formattable](formattable.html)<[ranges::range_reference_t](../../ranges/range_reference_t.html)<R>, CharT> is true. 

This specialization meets the [Formatter](../../named_req/Formatter.html "cpp/named req/Formatter") requirements if const R models [`input_range`](../../ranges/input_range.html "cpp/ranges/input range") and [ranges::range_reference_t](../../ranges/range_reference_t.html)<const R> models [std::formattable](formattable.html)<CharT>. It always meets the [BasicFormatter](../../named_req/BasicFormatter.html "cpp/named req/BasicFormatter") requirements.

### Format specification

The syntax of range-format-spec is:   
  
---  
range-fill-and-align ﻿(optional) width ﻿(optional) `**n**`(optional) range-type ﻿(optional) range-underlying-spec ﻿(optional) |  |   
  
The syntax is fully described in [range format specification](range_formatter.html#Range_format_specification "cpp/utility/format/range formatter"). 

For specializations of `std::formatter` where [std::format_kind](format_kind.html)<R> is either std::range_format::string or std::range_format::debug_string, the format-spec is std-format-spec instead of range-format-spec (which uses [std::formatter](formatter.html)<[std::basic_string](../../string/basic_string.html)<CharT>, CharT> as the underlying formatter).
