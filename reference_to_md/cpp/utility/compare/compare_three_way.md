##  std::compare_three_way::operator()

template< class T, class U >  
constexpr auto operator()( T&& t, U&& u ) const; |  |   
---|---|---  
| |   
  
Given the expression [std::forward](../forward.html)<T>(t) <=> [std::forward](../forward.html)<U>(u) as expr: 

  * If expr results in a call to [built-in](../../language/operator_comparison.html#Built-in_pointer_equality_comparison "cpp/language/operator comparison") operator<=> comparing pointers, given the [composite pointer type](../../language/pointer.html#Composite_pointer_type "cpp/language/pointer") of t and u as `P`: 



    

  * Compares the two converted pointers (of type `P`) in the [implementation-defined strict total order over pointers](../../language/operator_comparison.html#Pointer_total_order "cpp/language/operator comparison"): 



    

  * If t precedes u, returns [`std::strong_ordering::less`](strong_ordering.html "cpp/utility/compare/strong ordering"). 
  * If u precedes t, returns [`std::strong_ordering::greater`](strong_ordering.html "cpp/utility/compare/strong ordering"). 
  * Otherwise, returns [`std::strong_ordering::equal`](strong_ordering.html "cpp/utility/compare/strong ordering"). 



  * If the conversion sequence from `T` to `P` or the conversion sequence from `U` to `P` is not [equality-preserving](../../concepts.html#Equality_preservation "cpp/concepts"), the behavior is undefined. 



  * Otherwise: 



    

  * Returns the result of expr. 
  * If [std::three_way_comparable_with](three_way_comparable.html)<T, U> is not modeled, the behavior is undefined. 



This overload participates in overload resolution only if [std::three_way_comparable_with](three_way_comparable.html)<T, U> is satisfied. 

### [[edit](https://en.cppreference.com/mwiki/index.php?title=cpp/utility/compare/compare_three_way&action=edit&section=3 "Edit section: Example")] Example

Run this code
    
    
    #include <compare>
    #include <iostream>
     
    struct Rational
    {
        int num;
        int den; // > 0
     
        // Although the comparison X <=> Y will work, a direct call
        // to std::compare_three_way{}(X, Y) requires the operator==
        // be defined, to satisfy the std::three_way_comparable_with.
        constexpr bool operator==(Rational const&) const = default;
    };
     
    constexpr [std::weak_ordering](weak_ordering.html) operator<=>(Rational lhs, Rational rhs)
    {
        return lhs.num * rhs.den <=> rhs.num * lhs.den;
    }
     
    void print([std::weak_ordering](weak_ordering.html) value)
    {
        value < 0 ? [std::cout](../../io/cout.html) << "less\n" :
        value > 0 ? [std::cout](../../io/cout.html) << "greater\n" :
                    [std::cout](../../io/cout.html) << "equal\n";
    }
     
    int main()
    {
        Rational a{6, 5};
        Rational b{8, 7};
        print(a <=> b);
        print(std::compare_three_way{}(a, b));
    }

Output: 
    
    
    greater
    greater
