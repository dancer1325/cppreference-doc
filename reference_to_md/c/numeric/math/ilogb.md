1-3) Extracts the value of the unbiased exponent from the floating-point argument arg, and returns it as a signed integer value.

4) Type-generic macros: If arg has type long double, `ilogbl` is called. Otherwise, if arg has integer type or the type double, `ilogb` is called. Otherwise, `ilogbf` is called.

5) Expands to integer constant expression whose value is either [INT_MIN](../../types/limits.html "c/types/limits") or -[INT_MAX](../../types/limits.html).

6) Expands to integer constant expression whose value is either [INT_MIN](../../types/limits.html "c/types/limits") or +[INT_MAX](../../types/limits.html).

Formally, the unbiased exponent is the integral part of logr|arg| as a signed integral value, for non-zero arg, where `r` is [FLT_RADIX](../../types/limits.html "c/types/limits").

### [[edit](https://en.cppreference.com/mwiki/index.php?title=c/numeric/math/ilogb&action=edit&section=2 "Edit section: Return value")] Return value

If no errors occur, the unbiased exponent of arg is returned as a signed int value. 

If arg is zero, FP_ILOGB0 is returned. 

If arg is infinite, [INT_MAX](../../types/limits.html "c/types/limits") is returned. 

If arg is a NaN, FP_ILOGBNAN is returned. 

If the correct result is greater than [INT_MAX](../../types/limits.html "c/types/limits") or smaller than [INT_MIN](../../types/limits.html "c/types/limits"), the return value is unspecified and a domain error or range error may occur. 

### [[edit](https://en.cppreference.com/mwiki/index.php?title=c/numeric/math/ilogb&action=edit&section=3 "Edit section: Error handling")] Error handling

Errors are reported as specified in [`math_errhandling`](math_errhandling.html "c/numeric/math/math errhandling"). 

A domain error or range error may occur if arg is zero, infinite, or NaN. 

If the correct result is greater than [INT_MAX](../../types/limits.html "c/types/limits") or smaller than [INT_MIN](../../types/limits.html "c/types/limits"), a domain error or a range error may occur 

If the implementation supports IEEE floating-point arithmetic (IEC 60559), 

  * If the correct result is greater than [INT_MAX](../../types/limits.html "c/types/limits") or smaller than [INT_MIN](../../types/limits.html "c/types/limits"), [FE_INVALID](../fenv/FE_exceptions.html "c/numeric/fenv/FE exceptions") is raised. 
  * If arg is ±0, ±∞, or NaN, [FE_INVALID](../fenv/FE_exceptions.html "c/numeric/fenv/FE exceptions") is raised. 
  * In all other cases, the result is exact ([FE_INEXACT](../fenv/FE_exceptions.html "c/numeric/fenv/FE exceptions") is never raised) and [the current rounding mode](../fenv/FE_round.html "c/numeric/fenv/FE round") is ignored. 



### [[edit](https://en.cppreference.com/mwiki/index.php?title=c/numeric/math/ilogb&action=edit&section=4 "Edit section: Notes")] Notes

If arg is not zero, infinite, or NaN, the value returned is exactly equivalent to (int)[logb](logb.html)(arg). 

[POSIX requires](https://pubs.opengroup.org/onlinepubs/9699919799/functions/ilogb.html) that a domain error occurs if arg is zero, infinite, NaN, or if the correct result is outside of the range of int. 

POSIX also requires that, on XSI-conformant systems, the value returned when the correct result is greater than [INT_MAX](../../types/limits.html "c/types/limits") is [INT_MAX](../../types/limits.html "c/types/limits") and the value returned when the correct result is less than [INT_MIN](../../types/limits.html "c/types/limits") is [INT_MIN](../../types/limits.html "c/types/limits"). 

The correct result can be represented as int on all known implementations. For overflow to occur, [INT_MAX](../../types/limits.html "c/types/limits") must be less than [LDBL_MAX_EXP](../../types/limits.html) * log2([FLT_RADIX](../../types/limits.html)) or [INT_MIN](../../types/limits.html "c/types/limits") must be greater than [LDBL_MIN_EXP](../../types/limits.html) - [LDBL_MANT_DIG](../../types/limits.html)) * log2([FLT_RADIX](../../types/limits.html)). 

The value of the exponent returned by `ilogb` is always 1 less than the exponent retuned by [frexp](frexp.html "c/numeric/math/frexp") because of the different normalization requirements: for the exponent `e` returned by `ilogb`, |arg*r-e  
| is between 1 and `r` (typically between 1 and 2), but for the exponent `e` returned by [frexp](frexp.html "c/numeric/math/frexp"), |arg*2-e  
| is between 0.5 and 1. 

### [[edit](https://en.cppreference.com/mwiki/index.php?title=c/numeric/math/ilogb&action=edit&section=5 "Edit section: Example")] Example

Run this code
    
    
    #include <fenv.h>
    #include <float.h>
    #include <math.h>
    #include <stdio.h>
    // #pragma STDC FENV_ACCESS ON
     
    int main(void)
    {
        double f = 123.45;
        [printf](../../io/fprintf.html)("Given the number %.2f or %a in hex,\n", f, f);
     
        double f3;
        double f2 = [modf](modf.html)(f, &f3);
        [printf](../../io/fprintf.html)("modf() makes %.0f + %.2f\n", f3, f2);
     
        int i;
        f2 = [frexp](frexp.html)(f, &i);
        [printf](../../io/fprintf.html)("frexp() makes %f * 2^%d\n", f2, i);
     
        i = ilogb(f);
        [printf](../../io/fprintf.html)("logb()/ilogb() make %f * %d^%d\n", f/[scalbn](scalbn.html)(1.0, i), [FLT_RADIX](../../types/limits.html), i);
     
        // error handling
        [feclearexcept](../fenv/feclearexcept.html)([FE_ALL_EXCEPT](../fenv/FE_exceptions.html));
        [printf](../../io/fprintf.html)("ilogb(0) = %d\n", ilogb(0));
        if ([fetestexcept](../fenv/fetestexcept.html)([FE_INVALID](../fenv/FE_exceptions.html)))
            [puts](../../io/puts.html)("    FE_INVALID raised");
    }

Possible output: 
    
    
    Given the number 123.45 or 0x1.edccccccccccdp+6 in hex,
    modf() makes 123 + 0.45
    frexp() makes 0.964453 * 2^7
    logb()/ilogb() make 1.92891 * 2^6
    ilogb(0) = -2147483648
        FE_INVALID raised
